# Contributing to Awesome AI + GRC

Thank you for your interest in contributing! This document provides guidelines for adding resources to this list.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it before contributing.

## Adding to this List

Please ensure your pull request adheres to the following guidelines:

### Formatting Rules

- Use the format: `[Resource Name](link) - Description.`
- Use `-` for bullet points (not `*` or `+`)
- Descriptions should end with a period
- Links should not have trailing slashes
- Ensure descriptions are clear, concise, and objective

### Quality Standards

To maintain the quality and focus of this list, please ensure submissions meet these criteria:

**AI + GRC Intersection Required**
- Resources must be at the intersection of AI and GRC
- This includes:
  - AI-powered tools for GRC automation (compliance, risk, audit)
  - Frameworks and standards for governing AI systems
  - Training and certifications combining AI and GRC expertise
  - Templates, prompts, and scripts for AI+GRC use cases

**Maintenance Status**
- Resources must be actively maintained OR clearly marked as archived
- For archived resources, they must still provide significant historical or reference value
- Check that links work and content is accessible

**Practical Value**
- Resources must provide practical value to practitioners
- Avoid purely promotional or marketing content
- Academic papers should have practical applications or frameworks

**Licensing Clarity**
- Free resources are preferred
- Commercial resources must be clearly marked with pricing indication
- Open source projects should have clear license information

### Submission Process

1. Search previous suggestions before making a new one to avoid duplicates
2. Make an individual pull request for each suggestion
3. Use the following title format: `Add [Resource Name]`
4. In the PR description, explain why the resource is awesome and how it fits the AI+GRC intersection
5. New categories are welcome if they contain at least 3 resources

### Pull Request Guidelines

- Check your spelling and grammar
- Ensure your text editor removes trailing whitespace
- Make sure your addition is placed in the appropriate category
- The pull request should have a useful title and description
- Review your changes before submitting

## Updating Your PR

Sometimes, maintainers will ask you to edit your pull request before it can be included. This is normally due to spelling errors or because your PR didn't match the guidelines.

If you're not sure how to update your pull request, check out GitHub's [guide to changing a pull request](https://help.github.com/articles/changing-a-pull-request/).

## Thank You

Your contributions are greatly appreciated! Every addition makes this resource more valuable for the AI+GRC community.
